# Phase 6 · 검색 · 필터 설계서

> 메인 목록에 키워드 검색 + 카테고리·지역 필터를 더한다. 필터 상태는 URL 쿼리로 관리(공유·뒤로가기 자연스럽게).
> ⚠️ 실측 반영: `items.title`(NOT NULL)·`description`(nullable) 존재(title 엔 trigram GIN 인덱스),
>   `item_categories` 조인 테이블 존재, `regions` 3개 시 단위(`jeju`=제주시 / `seogwipo`=서귀포시 /
>   `all`=제주 전체(이동 가능)). `items.region_id` 는 이 중 하나를 가리킨다.

---

## 1. 목표
자재가 쌓여도 **키워드·카테고리·지역**으로 원하는 자재를 빨리 찾는다.
- 검색창은 메인 상단 항상 노출, 카테고리·지역은 '필터' 버튼으로 펼침(당근식)
- 거래종류 탭(기존) + 검색 + 필터를 **동시 적용**
- 모든 상태는 URL 쿼리(`?type=&q=&category=&region=`)

---

## 2. 사용자 시나리오
### A · 키워드로 찾기
1. 메인 상단 검색창에 "각재" 입력 → 엔터(또는 검색)
2. 제목·설명에 "각재"가 든 자재만 표시 (`?q=각재`)

### B · 카테고리·지역으로 좁히기
1. '필터' 버튼 → 패널 펼침 → 카테고리 1개 + 지역(제주시/서귀포시/제주전체) 선택 → 적용
2. 해당 조건 자재만 표시. 거래종류 탭·검색어와 함께 적용됨

### C · 초기화 / 결과 없음
- '초기화'로 필터·검색 해제 → 전체 목록
- 조건에 맞는 게 없으면 "검색 결과가 없어요"

---

## 3. 만들 파일 목록
| 경로 | 종류 | 책임 |
|---|---|---|
| `src/app/_components/search-bar.tsx` | Client | 검색 입력 → `?q=` 갱신(제출 시) |
| `src/app/_components/filter-panel.tsx` | Client | '필터' 버튼 + 펼침 패널(카테고리·지역 단일 선택) → `?category=&region=` 갱신, 초기화 |

**수정할 파일:**
- `src/app/page.tsx` — `searchParams`에 `q`/`category`/`region` 추가, 카테고리·지역 옵션(목록) 조회해 `filter-panel`에 전달, `item-list`에 필터 전달
- `src/app/_components/item-list.tsx` — `q`/`category`/`region` 필터를 쿼리에 반영 + "검색 결과 없음" 상태

---

## 4. 각 파일 책임 (핵심)
- **search-bar:** 입력 후 제출(엔터/버튼) → `?q=` 세팅(`useRouter`, 기존 쿼리 보존). 비우면 `q` 제거. (라이브 디바운스는 나중)
- **filter-panel:** '필터' 버튼 → 패널 펼침. 카테고리(단일) + 지역(제주시/서귀포시/제주전체). '적용' → `?category=&region=` 세팅 후 닫힘, '초기화' → 해제. 활성 필터 있으면 버튼에 표시(점/개수).
- **item-list 쿼리 반영:**
  - `q` → 제목·설명 부분일치: `.or("title.ilike.%q%,description.ilike.%q%")`. ⚠️ `q`의 `%`·`_`·`,`·`(`·`)`·`\` 등 특수문자는 이스케이프/치환해 `.or` 문법·와일드카드 깨짐을 막는다.
  - `category` → `item_categories!inner(category_id)` + `.eq` 임베드 필터 시도. 안 먹으면 `item_categories`에서 item_id 모아 `.in('id', ids)` 분리(실측 우선).
  - `region` → **지역 규칙(결정 1)**: 제주시/서귀포시 선택 시 해당 시 + `all`(제주 전체·이동 가능) 매물 포함, 제주전체 선택 시 `all` 만. `items.region_id` 가 단일 id 라, 해당 region id 들을 코드로 resolve 해 `.in('region_id', [...])` 로 거는 게 임베드 필터보다 robust.
  - 기존 정렬(거래완료 뒤로 + 최신순) 유지. 결과 0건이면 "검색 결과가 없어요".

---

## 5. 라우팅
신규 라우트 없음. 전부 `/`에서 쿼리(`?type=&q=&category=&region=`)로 처리. 거래종류 탭은 기존 `?type=` 그대로.

---

## 6. 디자인 토큰 활용
- 검색창·필터 버튼 — 흰 배경, primary 포인트(활성 필터 표시)
- 필터 패널 — 흰 카드/시트, 선택된 항목 primary 강조
- 결과 없음 — 깔끔한 안내 문구 + (선택)초기화 버튼

---

## 7. 예외 / 보안
- 검색·필터는 **공개 목록**이라 RLS(`items_select_all`) 그대로.
- `q` 사용자 입력은 `.or` 필터에 들어가니 특수문자 이스케이프(주입·문법 깨짐 방지).
- 잘못된 `category`/`region` 쿼리값 → 무시하고 전체(graceful).

---

## 8. 결정 사항 (6-A 진입 전 확정)

설계 9번 미결정 3개를 기본안으로 확정(2026-06-06).

1. **지역 필터 규칙** — 시(市) 선택 시 그 시 + `all`(제주전체) 매물 포함, 제주전체 선택 시 `all` 만.
2. **검색 트리거** — 제출(엔터/버튼). 라이브 디바운스는 나중.
3. **정렬 옵션**(가격순 등) — 이번 제외. 기존 거래완료 뒤로+최신순 유지.

- [ ] 거래종류 탭 + 검색 + 필터 **동시 적용**

- [ ] URL 쿼리로 상태 유지(공유·뒤로가기)
