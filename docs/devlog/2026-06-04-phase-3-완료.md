## 확정한 결정
1. 전화걸기 = 로그인 유저만 (`revealPhone` 인증 게이트, 번호 수확 방지)
2. 수정 = 사진까지 전체 편집 (삭제·추가·순서변경)
3. 거래완료 = 목록에 회색으로 계속 노출 (카드 dim)
4. 가격표기 = 금액+협의 병기 (판매중 50,000원 / 50,000원·협의 가능 / 나눔 무료 / 구해요 희망가 50,000원)

- 소유자 본인 글엔 전화하기 숨김

## 배운 점 / 메모
- **전화번호 보호:** 상세 쿼리에서 `contact_phone` 제외 → 초기 HTML·소스에 번호 없음. `revealPhone`만 서버 `getUser()` 통과 시 반환.
- **사진 순서변경:** `unique(item_id, display_order)` 제약 때문에 per-row UPDATE는 전이 충돌 위험 → 전체삭제+재삽입이 안전. 유지된 사진의 Storage 파일은 안 건드림.
- **삭제 정리:** DB CASCADE는 자식 행만 지움. Storage 파일은 별도 `remove()` 필요.
- **폼 단일 소스:** 공유 `ItemForm`으로 create/edit 검증·업로드 로직 drift 방지.
- **보안 원칙:** UI 숨김 ≠ 잠금. 가드(페이지) + 액션 양쪽에서 소유자 재검증.
- **비트랜잭션:** `updateItem` 사진 재삽입 구간이 추후 RPC 트랜잭션화 1순위(중간 실패 시 연결 전체 손실 위험).
- **커밋 교훈:** 묶음 단위로 바로 커밋해야 경계 안 섞임(이번에 4·5가 한 파일에 섞임).
