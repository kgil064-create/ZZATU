## 배운 점 / 메모
- ⭐ **Realtime + RLS 인증(이번 최대 함정):** RLS 켜진 테이블의 `postgres_changes`를 구독하려면 realtime 소켓이 유저 JWT로 인증돼야 함. `getSession()` → `supabase.realtime.setAuth(access_token)`를 `.subscribe()` 전에 호출. 빠지면 익명(`auth.uid()=null`)으로 붙어 RLS가 모든 이벤트 차단 → "F5 눌러야 뜸" 증상. 저장·초기로드는 서버 경로라 멀쩡해서 더 헷갈림.
- **chat-images 경로 = `{user_id}/{room_id}/파일명`(3단):** Storage 읽기 정책이 `path[2]=room_id`로 참여자 검증. item-images(2단)와 다름.
- **Private 버킷 사진:** `image_url`엔 경로만 저장, 표시 시점에 `createSignedUrl`(1시간).
- **같은 테이블 2개 FK 임베드**(`!buyer_id`/`!seller_id`)로 상대방 구분 + 마지막 메시지 임베드(referencedTable order/limit 1건) — 둘 다 동작 확인.
- **솔로 테스트법:** 둘째 카카오 없이 → 더미 사용자(대시보드 Add user) + 더미 자재(SQL) + SQL 메시지 insert로 Realtime·좌우 검증.
