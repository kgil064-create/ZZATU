# Phase 2 · 스키마 점검 일지

**날짜:** 2026-06-03
**단계:** Phase 2 작업 분할 1단계 — items 테이블 스키마 점검

## 결론
추가 마이그레이션 불필요. DB가 이미 Phase 2 설계서 요구사항을 충족함.

## 점검 결과

### 이미 준비된 것
- `items` 테이블 19개 컬럼 (id, user_id, type, title, price, price_option, region_id, region_memo, transport_options[], description, contact_phone, is_sold, view_count, created_at, updated_at 등)
- `item_categories` 조인 테이블 (다중 카테고리 지원, RLS 정책 3개 포함)
- `item_images` 테이블 (다중 사진, display_order로 순서 관리)

### DB에 있지만 Phase 2 폼에서는 안 받기로 한 컬럼
- `spec` (규격), `unit` (단위), `quantity` (수량), `condition` (상태)
- 이유: 설계서에서 "상세 설명 자유 텍스트 하나로 통합" 결정 (사용자 부담 최소화)
- DB엔 NULL로 남겨둠. 추후 사용자 피드백 보고 도입 가능성 열어둠.

### 5/30 셋업의 통찰
5/30에 만든 SQL 파일(01_create_tables.sql)에서 region_memo, item_categories 조인 테이블, item_images 등을 이미 만들어둬서, 설계서를 짜고 보니 추가 마이그레이션이 거의 필요 없는 상태였음. 셋업 단계에서 미래 요구사항을 잘 예측한 덕.
