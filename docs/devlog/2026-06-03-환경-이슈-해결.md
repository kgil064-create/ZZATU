# 환경 이슈 해결 일지

**날짜:** 2026-06-03
**맥락:** Phase 2 진행 중 발생한 환경 이슈들과 그 결정

## 발생한 문제
1. **Turbopack 워크스페이스 인식 실패** — Next.js 16의 Turbopack이 Dropbox 안 한글·공백 경로를 워크스페이스 루트로 인식 못함. `turbopack.root` 옵션 명시해도 무시됨.
2. **긴 경로 권한 이슈** — Windows 260자 제한 + pnpm 깊은 폴더 구조 (.pnpm/next@.../...) 콤보로 PowerShell Remove-Item, pnpm store prune 등이 EPERM 에러 반복.
3. **pnpm install이 깨진 파일 재생성 안 함** — "Already up to date" 라며 실제로는 `next` 실행파일이 빠진 상태가 풀리지 않음.

## 결정

### 1. Turbopack → Webpack 전환
- `package.json` dev/build 스크립트에 `--webpack` 플래그 추가
- 짜투 규모에선 Turbopack vs Webpack 체감 차이 미미 (시작 415ms vs 401ms)
- 일시적 절충, 추후 Next.js 17+ 또는 환경 개선 시 복귀 가능

### 2. 프로젝트 폴더 이동
- `C:\Users\kgil0\Dropbox\.../zzatu` → `C:\dev\zzatu`
- 이유: Dropbox + 한글·공백 경로 + Windows 260자 제한 + pnpm 깊은 구조 = 누적 권한 이슈
- 폴더 이동 + `pnpm install` 새로 = 모든 환경 문제 해소
- Dropbox 백업은 별도 전략 필요 (GitHub push가 1차 백업, Dropbox 별도 폴더 검토)

## 교훈
- Next.js + pnpm + Windows + Dropbox 콤보는 초기 셋업 단계에서 충돌 가능성 큼
- 환경 이슈에 매달리지 말고, **"지금 작업이 동작하는지"** 가 최우선
- 다음 프로젝트는 처음부터 `C:\dev\` 같은 단순 경로 권장
