## 배운 점 / 메모
- **닉네임 단일 소스:** 헤더·마이페이지 모두 `profiles.nickname`. `updateNickname`이 `revalidatePath("/", "layout")`로 헤더까지 갱신. (소스를 안 묶으면 수정이 헤더에 안 보이는 버그)
- **item_views:** `(user_id, item_id)` 복합 PK + upsert → 재조회 시 `viewed_at`만 갱신. FK cascade로 글/계정 삭제 시 자동 정리.
- **StrictMode 가드:** 개발 모드에서 effect가 두 번 도는 걸 ref로 막아 중복 기록 방지.
- **graceful degradation:** 테이블 미생성 상태에서도 최근 본 글이 빈 화면으로 떨어져 500 안 남.
- **솔직 교훈:** "본인 글 제외"는 깔끔해 보였지만 단일 계정에선 테스트가 불가능 → "포함"으로 전환. 설계안이 테스트 현실과 충돌하면 조정한다.
