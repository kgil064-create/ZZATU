# 카테고리·지역 단순화 일지

**날짜:** 2026-06-04
**맥락:** Phase 2 자재 등록 폼 4단계 picker 컴포넌트 시각 검토 후, 사용자 부담 줄이기 방향으로 정책 변경

## 변경 1: 카테고리 최대 선택 개수
- 변경 전: 최대 5개
- 변경 후: 최대 3개
- 이유: 자재 1개가 5개 카테고리에 걸친다면 "기타"로 분류하는 게 더 정확. 검색 노이즈 방지.

## 변경 2: 지역 입력 방식
- 변경 전: 43개 읍면동 단일 선택
- 변경 후: 3택 옵션 (제주시 / 서귀포시 / 제주 전체(이동 가능))
- 이유:
  - 43개 중 고르기 부담 (제주 잘 모르는 사람한텐 어려움)
  - "제주 전체(이동 가능)" 옵션이 자재 거래 현실 반영 — 작은 자재는 양쪽 어디든
  - 정확도(좁은 지역)는 검색·필터 기능 만들 때 다시 고려

## DB 변경 (가역적)
- `regions` 테이블에 3개 행 추가 (`display_order < 100` 으로 신구 구분)
- 기존 43개 읍면동 행은 보존, 폼에선 `.lt('display_order', 100)` 필터로 숨김
- CHECK 제약 수정: `si in ('jeju', 'seogwipo')` → `si in ('jeju', 'seogwipo', 'all')`
- 추후 세분화 필요 시 폼만 수정하면 기존 데이터 다시 활용 가능

## 관련 커밋
- `feat: 카테고리 3개 제한 + 지역을 3택(제주시/서귀포시/전체)으로 단순화`
